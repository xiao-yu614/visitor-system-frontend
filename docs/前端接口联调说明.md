# 前端接口联调说明

## 1. 环境配置

### 1.1 后端服务地址

前端默认配置的后端服务地址：
```
http://localhost:8080/api
```

### 1.2 环境变量配置

在项目根目录的 `.env` 文件中配置：

```env
VITE_API_BASE_URL=http://localhost:8080/api
```

### 1.3 启动后端服务

确保后端服务在 `8080` 端口运行：

```bash
# 进入后端项目目录
cd visitor-system-backend

# 启动后端服务
mvn spring-boot:run
```

## 2. API 接口清单

### 2.1 认证模块

| 接口 | HTTP方法 | 路径 | 需要登录 |
|------|---------|------|----------|
| 登录 | POST | `/auth/login` | ❌ |
| 登出 | POST | `/auth/logout` | ✅ |
| 获取当前用户 | GET | `/auth/user` | ✅ |

**登录请求示例**：
```json
POST /api/auth/login
{
  "username": "admin",
  "password": "123456"
}
```

**登录响应示例**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "xxx.xxx.xxx",
    "user": {
      "id": "1",
      "username": "admin",
      "name": "管理员",
      "role": "admin"
    }
  }
}
```

### 2.2 访客申请模块

| 接口 | HTTP方法 | 路径 | 需要登录 |
|------|---------|------|----------|
| 提交申请 | POST | `/visit/apply` | ❌ |

**请求示例**：
```json
POST /api/visit/apply
{
  "visitorName": "张三",
  "phone": "13800138000",
  "idCard": "110101199001011234",
  "company": "北京科技有限公司",

  "reason": "技术交流",
  "startTime": "2026-05-20 10:00:00",
  "endTime": "2026-05-20 12:00:00"
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "申请提交成功",
  "data": {
    "applyId": "AP20260520001"
  }
}
```

### 2.3 审批模块

| 接口 | HTTP方法 | 路径 | 需要登录 |
|------|---------|------|----------|
| 获取申请列表 | GET | `/admin/apply/list` | ✅ |
| 审批申请 | PUT | `/admin/apply/{id}/approve` | ✅ |

**获取申请列表请求**：
```
GET /api/admin/apply/list?status=pending&page=1&size=10
```

**审批请求示例**：
```json

{
```

## 3. 数据格式说明

### 3.1 日期时间格式

前端向后端传递的日期时间格式：
```
yyyy-MM-dd HH:mm:ss
```

示例：`2026-05-20 10:00:00`

### 3.2 状态码说明

| 状态码 | 说明 |
|--------|------|
| 200 | 成功 |
| 0 | 成功（备选） |
| 401 | 未登录或登录过期 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 500 | 服务器错误 |

## 4. Token 机制

### 4.1 Token 存储

前端将 Token 存储在 `localStorage` 中：
```javascript
localStorage.setItem('visitor_system_token', token)
```

### 4.2 Token 携带

前端通过请求头自动携带 Token：
```
Authorization: Bearer xxx.xxx.xxx
```

### 4.3 登录失效处理

当后端返回 401 状态码时，前端自动：
1. 清除 localStorage 中的 Token
2. 跳转至登录页

```typescript
// 使用模拟数据（当前配置）
```

## 7. 常见问题

### 7.1 跨域问题

如果出现跨域错误，请确保后端配置了 CORS：

```java
@Configuration
public class CorsConfig {
    @Bean
    public WebMvcConfigurer corsConfigurer() {
        return new WebMvcConfigurer() {
            @Override
            public void addCorsMappings(CorsRegistry registry) {
                registry.addMapping("/api/**")
                        .allowedOrigins("http://localhost:5173")
                        .allowedMethods("GET", "POST", "PUT", "DELETE")
                        .allowCredentials(true);
            }
        };
    }
}
```

### 7.2 Token 失效

如果 Token 失效，前端会自动跳转登录页。

### 7.3 接口返回格式

后端返回格式需包含 `code`、`message`、`data` 字段：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```
