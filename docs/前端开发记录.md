## 开发规范

### 代码风格

- 使用 2 空格缩进
- 使用单引号
- 变量命名使用 camelCase
- 组件命名使用 PascalCase

### Git 提交规范

- feat: 新增功能
- fix: 修复 bug
- docs: 文档更新
- style: 代码格式调整
- refactor: 代码重构
- test: 测试用例
- chore: 构建/工具更新
